# Правила работы с проектом «Бот записи»

## Общие правила
- Перед созданием нового файла — сначала расскажи, что планируешь
- Один универсальный бот, настройки клиента через config.json
- Для нового клиента — меняется только config.json, код не трогается

## Стек
- Python 3.11+
- aiogram 3 (async)
- SQLite через aiosqlite
- APScheduler для напоминаний
- python-dotenv для .env

## Структура проекта
```
zapis_bot/
├── bot.py           — точка входа
├── config.py        — загрузка .env + config.json
├── config.json      — услуги, цены, расписание (меняется под клиента)
├── database.py      — работа с БД
├── handlers/
│   ├── client.py    — хендлеры клиента
│   └── admin.py     — хендлеры мастера
├── keyboards.py     — inline-клавиатуры
├── scheduler.py     — напоминания
├── .env             — токен + ID (НЕ коммитить)
├── .env.example     — шаблон .env
└── requirements.txt — зависимости
```

## Правила кода
- Асинхронный код (async/await)
- Все тексты на русском
- Inline-клавиатуры (не reply)
- try/except с логированием — бот не должен падать
- Часовой пояс: Europe/Moscow

## Что НЕ делать
- Не хардкодить услуги/цены — всё через config.json
- Не коммитить .env и .db файлы
- Не использовать внешние API (бот работает без интернет-сервисов, кроме Telegram)
