# Правила работы с проектом «Продукт / Брошюра + Бот»

## Общие правила
- Перед созданием нового файла — сначала расскажи, что планируешь
- Брошюра пишется простым языком, как инструкция к мебели
- Без канцелярита, без сложных терминов
- После завершения задачи — всегда git commit + push

## Стиль текстов
- Обращение на «ты»
- Короткие предложения
- Каждый шаг — одно действие
- Чеклист после каждого шага
- Готовые шаблоны (копируй-вставляй)

## Формат брошюры
- Основной файл: БРОШЮРА.md
- HTML-версия: broshura.html (латинское имя для браузера)
- Кнопка «Скачать PDF» через window.print()

## Бот записи (product/bot/)
- Стек: Python 3.11+, aiogram 3, SQLite, APScheduler
- Токен и ID в .env (не коммитить!)
- Бот @Manic_zapis_bot — демо для показа клиентам

### Текущие возможности бота
- Запись клиента: услуга → дата → время → телефон → подтверждение
- Сбор телефона клиента при первой записи
- Уведомление мастеру при новой записи (со звуком)
- Напоминания клиенту: за 6ч, 2ч и 1ч до визита
- Кнопка «Добавить в календарь» — только для мастера
- Автосинхронизация с Google Календарь (нужен google_credentials.json)
- Отмена записи клиентом + уведомление мастера
- Команды мастера: /admin, /today, /tomorrow, /week, /stats

### Структура бота
```
bot/
├── bot.py              — основной файл, хендлеры
├── config.py           — загрузка .env
├── database.py         — SQLite, услуги, CRUD
├── calendar_sync.py    — синхронизация с Google Calendar
├── .env                — токен + ID (НЕ коммитить)
├── .env.example        — шаблон
├── requirements.txt    — зависимости
└── manicure.db         — БД (создаётся автоматически)
```

## Что НЕ делать
- Не обещать «пассивный доход» — честно: микробизнес с низкой нагрузкой
- Не занижать цены — минимум 5 000 ₽ настройка
- Не выдумывать конверсию — писать реальные цифры
- Не коммитить .env, .db, google_credentials.json
- Не показывать клиенту кнопку календаря — только мастеру

## Обновление PLAN.md
Когда пользователь отмечает задачу выполненной — обновить PLAN.md:
- Перенести из «В работе» / «Планы» → «Сделано» с [x]
- Добавить новые задачи если появились
