# Правила работы с проектом «Продукт / Брошюра»

## Общие правила
- Перед созданием нового файла — сначала расскажи, что планируешь
- Брошюра пишется простым языком, как инструкция к мебели
- Без канцелярита, без сложных терминов

## Стиль текстов
- Обращение на «ты»
- Короткие предложения
- Каждый шаг — одно действие
- Чеклист после каждого шага
- Готовые шаблоны (копируй-вставляй)

## Формат брошюры
- Основной файл: БРОШЮРА.md
- HTML-версия: БРОШЮРА.html (генерируется из .md)

## Шаблон бота (product/bot/)
- Должен соответствовать ТЗ из брошюры (шаг 3)
- config.json для настройки под клиента
- .env.example с пояснениями

## Что НЕ делать
- Не обещать «пассивный доход» — честно: микробизнес с низкой нагрузкой
- Не занижать цены — минимум 5 000 ₽ настройка (стартовый тариф)
- Не выдумывать конверсию — писать реальные цифры
- Не игнорировать конкурентов (YCLIENTS, Dikidi) — показывать УТП
