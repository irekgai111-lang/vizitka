# Правила работы с проектом «Визитка»

## Общие правила
- Все страницы делаем в одном HTML-файле, без внешних библиотек
- Перед созданием нового файла — сначала расскажи, что планируешь

## Дизайн
- Тёмный фон (#0a0a0a)
- Светлый текст (белый + rgba оттенки)
- Акцентный цвет: бирюзовый (#00d2c8)
- Адаптивная вёрстка — всё должно выглядеть хорошо на мобильных
- Шрифт: Segoe UI / system fonts

## Стиль текстов
- Дружелюбный, простой, без канцелярита
- На «вы» для учеников (официальный, но тёплый)

## Структура
- Основной файл визитки: `index.html` (самодостаточный: CSS в `<style>`, JS в `<script>`, SVG инлайн)
- Утилита `qr.html` — локальный генератор QR-кодов для публичного URL (использует CDN, это исключение из правила «без внешних библиотек», т.к. не часть визитки)
- Изображения: только logo.jpg

## Публикация
- Хостинг: GitHub Pages — `https://irekgai111-lang.github.io/vizitka/`
- Репозиторий: `github.com/irekgai111-lang/vizitka`
- Git-корень находится на уровне `my-first-project/` (визитка — подпапка)
- Флоу обновления: правим → коммитим **только файлы из папки `vizitka/`** → `git push` → GitHub Pages раздаёт новую версию через ~30 секунд
- При коммитах не трогать файлы из соседних папок (ai-tutor, landing, product, vocal-bot) без явной просьбы пользователя

## Что НЕ делать
- Не подключать внешние CSS/JS библиотеки в `index.html` (исключение — `qr.html`, см. выше)
- Не создавать отдельные CSS/JS файлы для визитки
- Не менять ссылки на соцсети без согласования
- Не удалять маркировку рекламы (ФЗ-38)
